# Milestone Plan V1

## Plan Ufku

Plan, 2026-04-21 baslangic tarihine gore sekillendirilmis kurumsal V1 yol haritasidir. Tarihler kontrollu varsayimdir; degisirse yeni DEC gerekmez ancak PM tarafindan delivery guncellemesi ve EVD etkisi islenir.

## Milestone'lar

| Milestone | Tarih Araligi | Hedef | Gate |
| --- | --- | --- | --- |
| M0 Initiation | 2026-04-21 - 2026-04-25 | Governance ve ORD/DEC cekirdegi | G0 |
| M1 Scope and IA Freeze | 2026-04-26 - 2026-05-08 | V1 kapsam, IA, curriculum cut sabitleme | G1 |
| M2 Architecture and Ops Freeze | 2026-05-09 - 2026-05-29 | ADR seti, Firestore modeli, security, ops standardi | G2 |
| M3 Build Ready | 2026-05-30 - 2026-06-12 | Repo scaffold, environment, emulator, seed, CI/CD | G3 |
| M4 Feature Delivery Core | 2026-06-13 - 2026-08-07 | Auth, onboarding, learning, gamification, social, shop | G4 |
| M5 Hardening and Release | 2026-08-08 - 2026-08-28 | Smoke, rollback, evidence, approvals, prod cikis | G5 |

## Milestone Kurallari

- Her milestone cikisi ilgili gate ile eslesir.
- Tarih kaymasi, kapsam degisikligi anlamina gelmez; kapsam degisiyorsa DEC gerekir.
- M4 icindeki work package kapanislari tek tek G4 mantigiyla kanitlanir.
